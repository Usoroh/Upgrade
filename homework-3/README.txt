Утилита `ping`:

Утилита `ping` используется для проверки доступности устройства по его IP-адресу или доменному имени. Утилита отправляет на указанный адрес запрос и ожидает от него ответ. Её можно использовать для разных целей:

- Проверка доступности устройства.
- Проверка связи.
- Диагностика проблем (можно посмотреть на каком участке маршрута происходит сбой).
- Трассировка маршрута (можно посмотреть по какому именно маршруту данные доставляются к точке назначения).
- Определение скорости соеденения (пинг показывает за какое время происходит отклик от устройства).

Результат работы команды `ping`:

- Количество отправленных пакетов.
- Количество полученных ответов.
- Количество потерянных пакетов (и процент потерь).
- Время в миллисекундах минимальное/максимальное/среднее/среднеквадратическое отклонение времени отклика.

Использование команды `curl -I https://kolesa.kz/`:

- HTTP/2 200 (протокол и код ответа).
- server: nginx (название сервера с которого пришел ответ).
- date: Fri, 14 Jul 2023 05:19:38 GMT (дата и время ответа).
- content-type: text/html; charset=UTF-8 (тип контента и кодировка).
- set-cookie: klssid=33f5mbq8k1bd4grcfl48ehs97a; expires=Fri, 14-Jul-2023 05:49:38 GMT; Max-Age=1800; path=/; domain=.kolesa.kz; secure; HttpOnly (кука со сроком годности и страницами которые ее принимают).
- expires: Thu, 19 Nov 1981 08:52:00 GMT (срок годности ответа).
- cache-control: no-store, no-cache, must-revalidate (параметры кэширования).
- pragma: no-cache (параметры кэширования для http 1).
- set-cookie: old_ssid=deleted; expires=Thu, 01-Jan-1970 00:00:01 GMT; Max-Age=0; path=/; secure (кука, протухает сразу??).
- set-cookie: is_returning=0; path=/; secure (кука, вероятно показывает был ли уже пользователь на сайте?).
- x-bug-bounty: Please report bugs and vulnerabilities to bugs@kolesa.kz (показывает адрес куда можно писать об уязвимостях).
- vary: Accept-Encoding, User-Agent (заголовки которые использовались для согласования содержимого).
- strict-transport-security: max-age=31536000; preload (показывает что с сайтом нужно общаться только через https).
- x-frame-options: SAMEORIGIN (позволяет отображать страницу в кадре на том же источнике, что и сама страница).
- content-security-policy: frame-ancestors 'self' https://webvisor.com (указывает допустимые источники которые могут встраивать страницу).
- alt-svc: h3=":443"; ma=2592000,h3-29=":443"; ma=2592000,h3-Q050=":443"; ma=2592000,h3-Q046=":443"; ma=2592000,h3-Q043=":443"; ma=2592000,quic=":443"; ma=2592000; v="46,43" (стороние сервисы и адреса с которых может подключится клиент).
